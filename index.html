System architecture

The Compute Brick is a modular stack. You can adopt one layer at a time without changing your phone.

Layers

1) Compute layer (Phone / Brick) ›
The phone becomes a docked compute core:
  • Belt clip / cross-body pouch / dock station
  • Optional passive cooling + cable strain relief
  • USB-C hub support: video out, power, peripherals
2) Display layer (AR / External) ›
Move the screen off your hands:
  • AR glasses for personal HUD
  • Projector/monitor for work + AV control
  • Dark-mode-first UI designed for low glare
3) Input layer (Ergo peripherals) ›
Inputs that match anatomy:
  • Lap split keyboard (thigh-resting halves)
  • Scroller wheel modules (index finger / edge wheel)
  • Ring / thumb controller for “mouse-like” control
  • Back paddles for quick actions
4) Software layer (UI + remapping) ›
A thin software layer ties it together:
  • Key remapping + macros
  • “Low stimulation” modes
  • Web-first control surfaces (TommyOS-ish)

Hardware concepts

Start with accessories that work today: ergonomic cases, scrollers, and split inputs. Then converge toward a fully open, modular compute brick.

Case geometry

Round-back “palm swell” case

Convex back + finger grooves + breathable grip. Reduces pinch grip and fingertip tension.

Scroll module

Side wheel / edge dial

Index-finger scrolling like a mouse wheel—no screen swiping needed. Great for reading and editing.

Quick actions

Back paddles

Two “middle finger” buttons for copy/paste, window switch, push-to-talk, AI key, etc.

Input end-game

Lap split ergo keyboard

Two halves resting on thighs for neutral wrists and shoulders. Optional strap/magnet mounts.

Pointer control

Ring / thumb controller

Lightweight pointer + clicks without a full mouse. Ideal for AR navigation.

Dock layer

Waist dock + USB-C hub

Cable management, cooling, quick release. Supports power, video out, and accessories.

Material direction

For a “natural computing” variant: prioritize tactile comfort, breathability, and repairability.

Grip materials
Textured biopolymer, cork insert, or hemp composite shell.
Lining
Linen/merino sleeve options for the waist dock (soft + breathable).
Modularity
Replaceable scroller/paddle modules, open fasteners, minimal glue.

Open-source & product strategy

Start with accessories (lowest friction), publish specs openly, then evolve toward a full “open computing” stack.

Open spec

Publish the interface, not just the object

Define the “inputs” (dial/paddles/keyboard) as HID events + open mappings so anyone can build compatible modules.

Business path

Sell the best hardware, keep the system open

Community can fork and improve. You win by shipping ergonomic quality + documentation + support.

Developer hook

“Phone as SBC” toolkit

Provide a web-first UI shell + templates for control surfaces (AV, notes, tasks, comms).

Sensory default

Low stimulation mode

Minimal animations, warm/dim presets, big typography, and a “calm” interaction model designed in from day one.
